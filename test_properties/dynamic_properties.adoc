## Как задать значение проперти для spring теста динамически

Как правило, когда вы используете в Spring Framework тестах какие-то специфичные properties,
у вас есть несколько вариантов чтобы задать их значения:

### application.properties

вы можете использовать файл application.properties в тестовых ресурсах, тогда эти настройки
будут применены ко всем тестовым сценариям которые вы пишите

[source, yaml]
----
application.properties
	sprint.redis.host = 127.0.0.1:6198
----

### TestPropertySource

можно указать значение свойств прямо в тест-кейсе при помощи аннотации TestPropertiesSource,
которая делает inline значения указаного в ней

[source, java]
----
@SpringBootTest
@ExtendWith(SpringExtension.class)
@TestPropertySource(properties = "spring.redis.host=127.0.0.1")
class SomeRedisTest {


}
----

Но что делать если значение которое вам нужно задать не константа,
в нашем примере хост и порт на котором стартует redis могут быть нам не известны
до того как мы запустим тест.
В такой ситуации легко оказаться, если вы например исопльзуете TestContainers.
До того как будет запущен контейнер, вы не сможете указать порт для теста.
Таким образом вариант использования application.properties отпадает автоматически,
как и аннотация TPS, потому что проставить значение внутри java аннотации можно только константное, мы не можем вызвать там метод.

Для пятого junit-а в библиотеке testcontainers есть аннотация `Testcontainers`, помогающая работать с контейнерами в тесте,
она ищет их в коде по наличию анотации `Container` и запускает выполнением.
Конечно можно было сделать решение заточенное специально для этой задачи, и добавить свой extension для junit5 которой бы проставлял на каком host/port запустился контейнер, но ведь можно сделать что-то более универсальное, ведь этот сценарий не единственный когда надо задать занчение проперти динамически для теста.

и вот что у меня получилось:

[source, java]
----
@SpringBootTest
@ExtendWith(SpringExtension.class)
class SimpleTest {

	@DynamicTestProperty <1>
	private static TestPropertyValues props() {
		return TestPropertyValues
				.of("variable=" + (int) Math.sqrt(64));   <2>
	}

	@Value("${variable}")   <3>
	private int variable;

	@Test
	void testSqrt() {
		assertThat(variable).isEqualTo(8);  <4>
	}
}
----
<1> при помощи аннотации DynamicTestProperty помечаем метод вычисляющий нам значение проперти
<2> этот метод должен возвращать TestPropertyValues в которых можно указать нужные нам настройки
<3> инжектим в тест значение проперти при помощи стандартного механизма спринга
<4> проверяем что значение проперти было выставлено при помощи DynamicTestProperty


И чтобы все это работало в ваших spring-boot проектах, нужна всего лишь одна зависимость:

[source, java]
----
<dependency>
    <groupId>com.jupiter-tools</groupId>
    <artifactId>spring-dynamic-property</artifactId>
    <version>0.1</version>
</dependency>
----
